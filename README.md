# Offensive Security Roadmap

A structured, beginner-friendly path into offensive security and penetration testing — built for AEC Coding Club members who want to go from zero to their first CTF, box, or bug bounty. Please note that the scope of this journey is strictly confined to ethics which includes finding security flaws, vulnerabilities and exploiting them with given authority. Seeking out of the scope is a punishable offense and you might attract someone you don't want.

## Why this exists

Most coding clubs cover web dev, app dev, ML, competitive programming — but cybersecurity, especially the *offensive* side, rarely gets a proper introduction. This repo is my attempt to fix that: a curated roadmap, cheat sheets, and writeups that lower the barrier to entry for anyone in the club curious about ethical hacking.

This isn't a finished product — it's a living document I'll keep expanding as I learn and as I (hopefully) get to run sessions on this as part of Core Committee.

## About me

I'm Chiranjib Saikia aka Chiron, a 3rd sem EE student at AEC and I'm currently working towards the **CPTS (Certified Penetration Testing Specialist)** certification from HackTheBox, focused on offensive security — web exploitation, Active Directory attacks, and privilege escalation. This repo documents both my own learning and resources I wish existed when I started. It all starts with the Linux OS and most specifically Kali Linux, loaded with 500+ penetration tools but again, Kali is a loaded gun, you need to know how to handle it and make the most out of it. As a hacker, you must know how machines talk to each other and what changes their daily routines which helps to identify major security flaws. Offensive security will consume your time and demand your patience, but having the right mindset turns that effort into mastery. Put in the hours, trust the journey, and the payoff in the end is worth every single sacrifice.


## Who this is for

Complete beginners with basic Linux/networking curiosity, all the way up to people ready to start their first CTF or certification path. No prior security experience assumed.

##  Stage 5: Hands-On Hacker!
This is where you get your hands on to some real hacking processes. Follow each resource for the best guide.

* **Step-01**: Create accounts on [HackTheBox](https://www.hackthebox.com/) and [TryHackMe](https://www.google.com/url?sa=t&source=web&rct=j&opi=89978449&url=https://tryhackme.com/&ved=2ahUKEwj4x5CPw-GVAxXk3jgGHVGZHeMQFnoECAwQAQ&usg=AOvVaw2gvAQndb0wWgMfXtgVVb0c)
  
* **Step-02**: Follow the Penetration Tester Path on [HTB](https://academy.hackthebox.com/path/preview/penetration-tester) and [THM](https://tryhackme.com/hacktivities). On HTB, the penetration tester path opens the gateway to the [CPTS EXAM](https://academy.hackthebox.com/preview/certifications/htb-certified-penetration-testing-specialist) on HTB which will land you many internship and job opportunities as a Junior Penetration Tester in Offensive Security.

Remember to complete labs on [HTB LABS](https://www.hackthebox.com/hacker/hacking-labs) and challenges on [THM](https://tryhackme.com/challenges). You become a good hacker only through experience and not simply by watching videos or referring to some books. Get some hands on practice to real hacking because that's what companies like to see (don't forget about the line I referred to you before). For the best reference, I will be providing you with the best youtube channels which got me started in this field. If you want to learn something more, just go to youtube and search for videos provided by these channels. Refer to the [OWASP page](https://owasp.org/) and study about vulnerabilities, how hackers exploit them. You can refer to [OWASP TOP 10](https://owasp.org/Top10/2025/) which is a regularly updated report outlining the 10 most critical security risks for web applications. Read reports and recent security flaws to keep yourself updated. The best folow-up would be to follow [HackerNews](https://thehackernews.com/) and make sure to subscribe to them so you get the latest news beforehand.

## Youtube Channels you must follow:
 - **[IppSec](https://www.youtube.com/@ippsec)**: The gold standard for HTB writeups/walkthroughs. Essential for anyone doing HTB or prepping for OSCP/CPTS.
 - **[John Hammond](https://www.youtube.com/@_JohnHammond)**: CTFs, malware analysis, and general hacking content, very approachable for beginners.
 - **[NetworkChuck](https://www.youtube.com/@NetworkChuck)**: My personal favorite when I was a beginner. Great entry point for networking + basic hacking concepts, high production value, keeps beginners engaged.
 - **[TCM Security](https://www.youtube.com/@TCMSecurityAcademy)**: If you are a serious learner in cybersecurity, refer to this academy as they will teach you many valuable skills.
 - **[HackerSploit](https://www.youtube.com/@HackerSploit)**: Another serious person. Watch his playlist on Red Teaming Essentials to give you an insight into how real hackers find vulnerabilities in corporate networks.
 - **[STOK](https://www.youtube.com/@STOKfredrik) and [NahamSec](https://www.youtube.com/@NahamSec)**: Bug-bounty focused, web-app hacking and real-world mindset. Not easy enough to understand if you are a beginner so begin bug bounty only after knowing how to find vulnerabilities in real systems.
 - **[Ryan John](https://www.youtube.com/@ryan_phdsec/videos)**: Another great place if you are a beginner.
 - **[PwnFunction](https://www.youtube.com/@PwnFunction)**: Short, visual explanations of web vulns (XSS, CSRF, etc.) — great for building intuition.
